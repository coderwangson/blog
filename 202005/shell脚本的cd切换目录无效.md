# shell脚本的cd切换目录无效

我写了如下的shell脚本：

```
#!/bin/bash
cd ~
```  
但是通过`bash test.sh`之后我的当前目录并没有切换，这就很迷惑，为了看我在shell执行的时候是否切换了，我修改了代码如下：

```
#!/bin/bash
cd ~
echo $(pwd)
```  

执行后输出的确实是home目录，这就说明我其实在shell里面是执行了的，但是为啥在终端没有效果呢。后来翻了资料，发现其实用bash或者./test.sh执行的方式是类似与我们平时代码里面的函数，我们在终端里面输入`bash test.sh`其实相当于调用了test.sh这个函数，系统会另外开启一个终端,然后执行完后，友回到了之前的终端，所以你在我们当前终端看不到效果。  

如果你想直接执行，那么可以通过`source test.sh`或者`../test.sh`来执行，这样就会立刻起作用了。  

source命令又称为点命令，作用：在当前shell环境下读取并执行脚本中的命令，通常用于重新执行刚修改过的初始化文件，使之立即生效，而不必注销并登录。
